How This Site is Deployed to GitHub Pages

  1. Project Setup: The project is built with React, TypeScript, Vite, Tailwind CSS, and Shadcn UI. All source code is in the ai folder.
  2. Vite Configuration: The vite.config.ts file sets base: '/ai/' so that all assets and routes work under the /ai/ subpath, matching the GitHub Pages URL.
  3. React Router: The BrowserRouter in App.tsx uses basename="/ai" to ensure client-side routing works on GitHub Pages and in local preview.
  4. GitHub Actions Workflow: A workflow file in .github/workflows/deploy.yml automatically builds and deploys the site to the gh-pages branch whenever the main branch is updated.
  5. Build & Preview: Run npm run build to generate the static site in the dist folder. Use npm run preview to test locally at http://localhost:4173/ai/.
  6. GitHub Pages Settings: In the repository settings, GitHub Pages is configured to serve from the gh-pages branch and the /ai/ subpath.
  7. Result: The site is live at https://<your-github-username>.github.io/ai/ and updates automatically on every push to the main branch.

This setup ensures a modern, fast, and fully automated deployment workflow for static sites using Vite and React on GitHub Pages.

Why Do We Use npm run build and npm run preview? What is the ai Prefix?

  • Why npm run build? This command generates an optimized, static version of your site in the dist folder. It bundles, minifies, and prepares your app for production, just like it will be served on GitHub Pages. You must run this before deploying or previewing the production build.
  • Why npm run preview? This command starts a local server that serves the built dist folder. It lets you test your site exactly as it will appear in production, including all routing and asset paths. This is important because the development server (npm run dev) does not always match production behavior, especially with custom base paths.
  • Why the ai Prefix? The ai prefix is set in both vite.config.ts (base: '/ai/') and App.tsx (basename="/ai"). This ensures that all routes and assets are served under the /ai/ subpath, matching your GitHub Pages URL (https://<your-github-username>.github.io/ai/). Without this, navigation and asset loading would break when deployed to a subfolder.

In summary: npm run build prepares your site for deployment, npm run preview lets you test the real production build locally, and the ai prefix ensures everything works correctly on GitHub Pages.
